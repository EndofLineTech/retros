# Retros

A shared, public corpus of end-of-session retrospectives from projects run with the [claude-agent-dev-team](https://github.com/MotWakorb/claude-agent-dev-team) skills. Retros are the stress test for the skill system — patterns found here drive rule changes.

## Contributing

Use the `/retro` skill at the end of a session, then `/retro-sync` to push here (it forks and opens a PR if you don't have write access).

## Anonymization

Retros are pseudonymized before publishing: project names become stable `project-a` style pseudonyms, and a scrub pass redacts emails, IPs, credentials, and contributor-supplied denylist terms. Do not submit retros containing customer data, PII, or real project/client names — the scrub is a backstop, not the control.
